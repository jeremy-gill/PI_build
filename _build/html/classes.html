exception PI_build.build_tools.NoCompatibleTagDefined[source]¶

Raised when there are not correctly formatted tags in the git repository.

At least one tag of the form v0.0.0 should be defined.

class PI_build.build_tools.PIVersionInfo[source]¶

Encapsulates a project’s version info.

Used internally - you shouldn’t need to create one directly.

VER_PRODUCT_MAJOR¶

The project’s major version number

VER_PRODUCT_MINOR¶

The project’s minor version number

VER_PRODUCT_REVISION¶

The project’s revision number

VER_PRODUCT_BUILD¶

The project’s build number

SHORT_SHA1¶

A shortened SHA1 digest representation for the software

FULL_VERSION¶

The full version string

VERSION¶

A shortened version string

MAINTAINER¶

Name of the maintainer of the project

MAINTAINER_EMAIL¶

E-mail of the maintainer

TAG¶

Closest tag name

get_dictionary()[source]¶

Returns a dictionary of version info.

Call this method if you need to perform string replacements. The contents of the dictionary can be used for automatic string replacments, such as in the following example:

"full_version = %(FULL_VERSION)s" % build_tools.PIVersionInfo().get_dictionary()
Return type:A dictionary of version strings
PI_build.build_tools.get_version_strings(_dir)[source]¶

A global function that returns a PIVersionInfo object, encapsulating your project’s version info.

Parameters:_dir – directory of the toplevel folder in your project. .git should be found immediately below this directory.
Return type:A PIVersionInfo object.
